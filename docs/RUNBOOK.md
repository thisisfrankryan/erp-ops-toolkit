# ERP 运维脚本执行 Runbook

这份 Runbook 用于规范 ERP 实施、运维、技术支持场景中的脚本执行流程。

## 执行前

1. 确认问题来源：客户反馈、系统日志、数据库记录、前端截图、操作路径。
2. 明确影响范围：单个单据、单个商品、单个用户，还是批量数据。
3. 确认业务口径：状态码含义、库存实际数量、审批流程节点、财务对账结果。
4. 在测试环境复现或模拟执行。
5. 准备回滚方案和数据备份。

## 执行中

1. 先执行 SELECT，确认目标数据唯一。
2. 执行备份 SQL，保留修改前快照。
3. 开启显式事务。
4. 执行 UPDATE / INSERT。
5. 检查影响行数。
6. 影响行数符合预期才 COMMIT；否则 ROLLBACK。

## 执行后

1. 再次 SELECT 验证结果。
2. 让业务用户复核页面状态。
3. 记录执行时间、执行人、脚本名称、影响范围、执行结果。
4. 整理问题原因和预防建议。

## 场景检查清单

### 审批流 / 单据状态修正

1. 确认单据编号、当前状态、目标状态和业务审批结果。
2. 检查是否存在未完成的流程任务或待办记录。
3. 备份单据主表和相关流程表记录。
4. 仅修正明确卡住的目标记录，执行后让业务用户复核页面流转状态。

### 库存异常修正

1. 核对商品编码、仓库、批次、账面库存和实际库存。
2. 优先确认是否存在未过账单据、重复出入库或接口重复推送。
3. 备份库存表和相关出入库明细。
4. 修正后复核库存台账、出入库报表和财务对账影响。

### 用户权限复制

1. 确认源用户与目标用户属于同岗位或同职责范围。
2. 先查询源用户有效权限，确认目标用户没有残留权限。
3. 备份源用户和目标用户权限记录。
4. 事务内复制权限，行数必须与源用户有效权限数一致。

### 主数据去重

1. 先明确唯一性口径，例如供应商名称、税号、物料编码或银行账号。
2. 先预览重复组，并由业务确认保留哪一条记录。
3. 禁止直接 DELETE，优先使用 `is_active`、`duplicate_flag`、`duplicate_keep_id` 等字段标记。
4. 处理后复核引用关系，避免影响历史订单、库存和财务凭证。

### 财务月结阻塞 / 死锁诊断

1. 先确认影响范围：单个报表、某个部门，还是整个财务模块。
2. 查询阻塞链、长事务、锁对象和正在执行的 SQL 文本。
3. 保存诊断结果，确认阻塞源是否为卡死任务、报表任务或仍在运行的业务操作。
4. 未经业务负责人和 DBA 确认，不直接终止会话。
5. 处理后复核页面响应、任务状态、凭证生成结果和数据库阻塞情况。

### 批量政策更新

1. 先明确政策口径：生效日期、业务类型、状态范围、旧值和新值。
2. 先 SELECT 预览影响行数，并由业务负责人确认范围。
3. 备份所有受影响记录，事务内更新，行数不一致立即回滚。
4. 执行后抽样复核页面数据、报表数据和后续审批 / 结算链路。
