# ERP 实施运维高频场景复盘

本文档用于沉淀 ERP 实施、运维与技术支持中常见的客户工单场景。重点不是堆脚本，而是把问题处理闭环写清楚：先复现、再定位、再备份、再处理、最后复核和回访。

## 二八法则判断

ERP 实施运维工作中，大量客户工单并不是复杂研发问题，而是集中在少数高频场景：

- 用户侧：登录、权限、页面报错、查询导出、表单联动。
- 业务侧：字典、附件、接口回调、个性化配置。
- 数据侧：先查影响范围，先留备份，再做事务修正。
- 交付侧：记录服务台账，沉淀原因、脚本、执行结果和预防建议。

## 场景处理讲法

### 11. 页面白屏 / 按钮无响应

1. 让客户提供页面路径、操作步骤、截图和发生时间。
2. 加载 `app_error_recorder.js`，记录 JS error、Promise error、HTTP error、network error。
3. 导出浏览器本地错误记录，判断是前端脚本异常、接口返回异常，还是网络中断。
4. 将错误类型、接口地址、状态码、时间写入服务台账。

原理：客户口述通常不完整，前端错误记录能把“点了没反应”变成可追踪证据。

### 12. 大范围查询 / 导出拖慢系统

1. 确认客户查询条件：组织、项目、日期、状态。
2. 用 `query_export_guard.js` 要求选择日期范围和关键组织条件。
3. 超过建议范围时提示分批导出。
4. 如果页面仍慢，再结合 `report_query_diagnosis.sql` 看慢 SQL 和缺失索引。

原理：ERP 报表慢常常不是单点故障，而是没有条件约束导致扫描范围过大。

### 13. 组织切换后项目 / 仓库下拉仍是旧值

1. 复现客户操作：先选组织 A，再选项目 A1，然后切到组织 B。
2. 用 `dependent_select_reset.js` 在父级字段变化时清空项目、仓库等子级字段。
3. 重新根据父级值加载子级选项。
4. 提交前确认组织、项目、仓库属于同一业务范围。

原理：联动字段的核心是防止“旧缓存 + 新父级”的不一致组合进入后端。

### 14. 账号被锁 / 登录失败

1. 确认用户身份、登录名、部门和是否本人申请。
2. 用 `user_login_unlock.sql` 查询账号状态、锁定字段和失败次数。
3. 备份用户记录，只重置锁定状态、失败次数和锁定时间。
4. 让用户重新登录，并记录执行人和复核结果。

原理：登录问题不能顺手改角色和权限，只处理锁定字段，边界越清楚风险越低。

### 15. 看得到菜单但看不到数据

1. 确认用户是否能看到菜单、按钮，以及看不到的是哪类业务数据。
2. 用 `user_data_scope_audit.sql` 查询角色绑定和组织数据范围。
3. 找同岗位已确认正常的参考用户做对比。
4. 目标用户没有残留数据范围时，备份后复制参考数据范围。

原理：菜单权限决定“能不能进页面”，数据范围决定“能不能看到数据”，两者要分开排查。

### 16. 下拉框为空 / 字典值错误

1. 确认页面字段、字典类型、缺失的字典编码和显示名称。
2. 用 `dictionary_option_repair.sql` 查询当前字典项。
3. 若字典项被停用，备份后恢复启用；若缺失，备份后补齐。
4. 刷新缓存或重新登录验证页面下拉选项。

原理：ERP 里很多页面下拉依赖字典表，字典错了会表现成页面功能异常。

### 17. 报表 / 列表查询慢

1. 记录慢的页面、查询条件、时间段和用户。
2. 用 `report_query_diagnosis.sql` 查看相关慢 SQL、逻辑读、执行次数。
3. 查看缺失索引建议和现有索引。
4. 和开发 / DBA 确认是否改查询条件、补索引或调整报表口径。

原理：慢查询先定位瓶颈，不能一上来改数据；高逻辑读、无索引、函数包裹索引列都是常见原因。

### 18. 附件上传后看不到 / 串单

1. 确认业务单据号、附件名、上传时间和操作人。
2. 用 `attachment_orphan_audit.sql` 查询附件元数据与业务单据关联。
3. 对孤儿附件先备份，再标记为 `ORPHAN`，不直接删除。
4. 复核目标单据的附件列表和历史附件记录。

原理：附件通常由文件存储和数据库元数据共同决定，排查时要先看元数据关联。

### 19. OA / 移动审批回调失败

1. 确认接口编号、业务单号、回调时间和错误信息。
2. 用 `integration_retry_queue.sql` 筛选 `FAILED` 且重试次数未超过阈值的消息。
3. 备份消息队列记录，重置为 `PENDING` 并设置下一次重试时间。
4. 观察接口任务是否成功消费，避免重复回调同一业务。

原理：接口失败要区分临时网络失败和业务数据错误，只有前者适合受控重试。

### 20. 列表字段消失 / 页面布局异常

1. 确认是否只有单个用户异常，还是所有用户异常。
2. 若只有单个用户异常，用 `user_view_config_reset.sql` 查询个人页面配置。
3. 备份个人配置后，将个人配置失效，恢复系统默认视图。
4. 让用户刷新页面，确认列、查询条件和按钮恢复正常。

原理：很多 B/S ERP 支持个人列配置，配置损坏时看起来像系统故障，本质是用户级配置问题。
